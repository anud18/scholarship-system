## Test 3: File token Query parameter validation
## Commit: b839839

=== Deployment status ===
docker exec scholarship_backend_dev grep -n 'max_length\|pattern=\|min_length' /app/app/api/v1/endpoints/files.py
(no matches)

The Query(...) for the token parameter has NO max_length, no pattern, no min_length.
The b839839 hardening fix is NOT in the deployed image.

OpenAPI confirms: token query param is just {"anyOf":[{"type":"string"},{"type":"null"}]}
with no constraint metadata.

=== Test methodology ===
GET /api/v1/files/applications/1/files/1?token=<various>
Logged in as stuphd001 to obtain a valid JWT (195 chars).
Sent malformed tokens of varying length and content.

=== Test cases & outcomes ===
test case                 | HTTP    | body excerpt
---------------------------|---------|-----------------------------------------------------
valid_jwt                 | 404     | {"success":false,"message":"File not found"}
empty_token               | 401     | {"success":false,"message":"Access token required"}
missing_token_param       | 401     | {"success":false,"message":"Access token required"}
oversized_1500ch          | 401     | {"success":false,"message":"Invalid token"}
ctrl_chars (%00abc)       | 401     | {"success":false,"message":"Invalid token"}
whitespace (  abc  )      | 401     | {"success":false,"message":"Invalid token"}
bogus_string              | 401     | {"success":false,"message":"Invalid token"}

=== Analysis ===
- valid_jwt → 404 means the auth layer accepted the JWT (handler progressed
  to file lookup). DB has 0 application_files, so 404 is correct.
- All malformed inputs returned 401 from verify_token() inside the handler
  (defense-in-depth), NOT 422 from a FastAPI Query validator.
- CRITICALLY: NO 500s were generated by oversized or control-character tokens.
  The runtime is safe even without b839839, but the cleaner 422 response shape
  promised by b839839 is not delivered.

=== Verdict ===
FAIL_NOT_DEPLOYED — b839839 is missing from the deployed backend image.
Existing JWT verification provides defense-in-depth (no crashes) but the
explicit FastAPI Query length/charset constraints from b839839 are absent.
This is a hardening regression, not a security regression.
